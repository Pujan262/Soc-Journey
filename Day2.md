Day 2 — OSI Models

1.Physical Layer — Layer 1

The Physical layer is responsible for transmitting raw bits (0s and 1s) between devices. Anything that physically moves data from one device to another belongs here.
Examples: Ethernet cables, Fiber optic cables, WiFi signals

2.Data Link Layer — Layer 2

The Data Link layer handles communication between devices on the same local network. It uses MAC addresses to identify devices and interacts with the Physical layer to send and receive data.
Examples: Ethernet, MAC addresses, WiFi

3.Network Layer — Layer 3

The Network layer handles IP addresses and routing. It finds the best path for packets to travel from your device to the destination using IP addresses.
Examples: IP address, Routers

4.Transport Layer – Layer 4

The Transport layer breaks data into segments and ensures it arrives correctly from one side of the network to the other. 
TCP — reliable, checks everything arrived correctly 
UDP — fast, does not check, used for video calls and gaming

5.Session Layer – Layer 5

The Session layer opens, manages, and closes connections between two devices. It keeps the conversation alive as long as needed. Example: staying logged into your banking app Pratical banking app or website it log in as long as we need and log out 

6.Presentation Layer – Layer 6

The Presentation layer translates data into a format the application can understand. It also handles encryption and decryption. 
Example: HTTPS encrypting your data using SSL/TLS

7.Application Layer – Layer 7

The Application layer is what the user sees and interacts with directly. When you open a browser and visit google.com that is Layer 7. 
Examples: HTTP, DNS, FTP, SMTP
